# BicycleMES 数据库设计文档

> 基于 RuoYi-Vue (Spring Boot 4 + Vue 3) 二次开发  
> 数据库：MySQL 8.0 | 库名：ryvue | 字符集：utf8mb4  
> 总计 31 张表（若依系统 23 张 + MES 业务 8 张）

**关系说明：**

| 关联 | 说明 |
|------|------|
| 工单 → 物料 | `material_id` 引用物料表，指定生产什么车 |
| 工单 → 产线 | `line_id` 引用产线表，指定在哪条线生产 |
| 工单明细 → 工单 | `order_id` 引用工单表，一个工单拆多道工序 |
| 工单明细 → 工序 | `process_id` 引用工序表，每行对应一道工序 |
| 报工 → 工单 | `order_id` 引用工单表，报工数据关联到具体工单 |
| 质检 → 工单明细 | `detail_id` 引用工单明细，质检针对每道工序 |
| 质检 → 缺陷 | `defect_id` 引用缺陷类型表，记录不良原因 |

### 4.5 mes_work_order（生产工单）✍️ 手写状态机

| 字段 | 类型 | 键 | 说明 |
|------|------|-----|------|
| id | BIGINT(20) | PK | 主键 |
| order_no | VARCHAR(64) | UK | 工单号，如 WO-20260701-001 |
| material_id | BIGINT(20) | INDEX, FK→mes_material | 产品物料ID |
| quantity | INT(11) | | 计划生产数量 |
| completed_quantity | INT(11) | | 已完成数量 |
| line_id | BIGINT(20) | INDEX, FK→mes_production_line | 产线ID |
| plan_start_date | DATETIME | | 计划开始日期 |
| plan_end_date | DATETIME | | 计划结束日期 |
| actual_start_date | DATETIME | | 实际开始日期 |
| actual_end_date | DATETIME | | 实际结束日期 |
| status | VARCHAR(16) | INDEX | **状态机字段** |
| priority | VARCHAR(16) | | 优先级 LOW/MEDIUM/HIGH/URGENT |
| + 通用字段 | | | |

**状态机流转**（手写核心逻辑）：

```
                    ┌──────────┐
                    │   NEW    │  新建工单
                    └────┬─────┘
                         │ startWorkOrder()
                         ↓
                    ┌──────────┐
          ┌─────────│PRODUCING │  生产中（可报工）
          │         └────┬─────┘
          │              │ completeWorkOrder() → 满产触发
          │              ↓
          │         ┌──────────┐
          │         │INSPECTING│  质检中
          │         └────┬─────┘
          │              │ finishWorkOrder()
          │              ↓
          │         ┌──────────┐
          │         │COMPLETED │  已完成
          │         └──────────┘
          │
          │  任一状态（除 COMPLETED）→ cancelWorkOrder()
          │         ↓
          │   ┌──────────┐
          └──→│CANCELLED │  已取消
              └──────────┘
```

**业务约束**：
- 只有 NEW 可开始生产
- COMPLETED 不可退回任何状态
- COMPLETED 不可取消
- 满产（completed_quantity ≥ quantity）时自动转 INSPECTING（在报工服务中触发）

| 字段 | 类型 | 键 | 说明 |

| inspection_type | VARCHAR(32) | | 检验类型（来料检验/过程检验/成品检验） |
| defect_id | BIGINT(20) | INDEX, FK→mes_defect | 缺陷类型ID |
| inspection_result | VARCHAR(16) | INDEX | 检验结果 PASS/FAIL |
| defective_quantity | INT(11) | | 不良数量 |
| inspector | VARCHAR(64) | | 检验人 |
| inspection_time | DATETIME | | 检验时间 |
| description | VARCHAR(512) | | 检验描述 |
| + 通用字段 | | | |

**手写关联查询**（`QualityInspectionMapper.xml` — 4 表 LEFT JOIN）：

```
mes_quality_inspection
  LEFT JOIN mes_work_order_detail ON detail_id = mes_work_order_detail.id
  LEFT JOIN mes_work_order ON mes_work_order_detail.order_id = mes_work_order.id
  LEFT JOIN mes_material ON mes_work_order.material_id = mes_material.id
  LEFT JOIN mes_defect ON defect_id = mes_defect.id
```

**查询结果扩展字段**（非持久化，仅前端展示）：
- `orderNo` — 工单号（从 mes_work_order 联查）
- `processName` — 工序名（从 mes_process 联查）
- `defectName` — 缺陷名（从 mes_defect 联查）
- `materialName` — 物料名（从 mes_material 联查）

> ⚠️ 生成器只做单表 CRUD，4 表 LEFT JOIN 必须手写 resultMap + SQL。

### 4.8 mes_production_report（报工记录）✍️ 全手写

| 字段 | 类型 | 键 | 说明 |
|------|------|-----|------|
| id | BIGINT(20) | PK | 主键 |
| order_id | BIGINT(20) | INDEX, FK→mes_work_order | 工单ID |
| qualified_quantity | INT(11) | | 合格数量 |
| defective_quantity | INT(11) | | 不良数量 |
| reporter | VARCHAR(64) | | 报工人 |
| report_time | DATETIME | | 报工时间 |
| + 通用字段 | | | |

**手写业务逻辑**（`ProductionReportServiceImpl.addReport()`）：

```
@Transactional
public int addReport(ProductionReport report) {
    1. 状态校验：工单状态必须是 PRODUCING 或 INSPECTING
    2. 数量校验：合格 + 不良 ≤ 计划数量 - 已完成数量（防超报）
    3. INSERT 报工记录
    4. UPDATE 工单累计已完成数量（completed_quantity += qualified）
    5. 满产自动送检：completed_quantity ≥ quantity → status='INSPECTING'
}
```

**业务约束**：
- 非 PRODUCING/INSPECTING 状态不可报工
- 超量报工被拒绝
- 满产时自动触发状态流转
- @Transactional 保证 INSERT + UPDATE + 状态变更的原子性
